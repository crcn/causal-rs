# v0.11.0 Backend API Design

## Overview

This plan refines the v0.11.0 backend abstraction incorporating 5 critical recommendations:

1. **Serve lifecycle** - Explicit `CancellationToken` for graceful shutdown
2. **Capability traits** - Compile-time feature detection (no runtime errors)
3. **Backend-agnostic** - Simple core trait, implementation details internal
4. **Structured config** - Clear configuration over positional parameters
5. **Migration strategy** - Explicit clean break with a clear upgrade path

## Core API

### Backend Trait

```rust
#[async_trait]
pub trait Backend: Send + Sync + 'static {
    /// Backend name for diagnostics/logging.
    fn name(&self) -> &'static str {
        "custom"
    }

    /// Persist/publish a dispatched event.
    async fn publish(&self, event: DispatchedEvent) -> Result<()>;

    /// Start serving (blocks until shutdown signal received).
    async fn serve<D>(
        &self,
        executor: Arc<JobExecutor<D>>,
        config: BackendServeConfig,
        shutdown: CancellationToken,
    ) -> Result<()>
    where
        D: Send + Sync + 'static;
}

#[derive(Debug, Clone)]
pub struct BackendServeConfig {
    pub event_workers: usize,
    pub handler_workers: usize,
    pub event_worker: EventWorkerConfig,
    pub handler_worker: HandlerWorkerConfig,
    pub graceful_shutdown_timeout: Duration,
}

#[derive(Debug, Clone)]
pub struct EventWorkerConfig {
    /// Polling interval when no events available.
    pub poll_interval: Duration,
    /// Maximum hop count before DLQ (infinite loop detection).
    pub max_hops: i32,
    /// Maximum number of events a handler may emit in one batch.
    pub max_batch_size: usize,
    /// Maximum retry count for inline handler failures.
    pub max_inline_retry_attempts: i32,
}

#[derive(Debug, Clone)]
pub struct HandlerWorkerConfig {
    /// Polling interval when no handlers ready.
    pub poll_interval: Duration,
    /// Default timeout for handler execution.
    pub default_timeout: Duration,
    /// Maximum number of events a handler may emit in one batch.
    pub max_batch_size: usize,
}
```

### Capability Traits

```rust
/// Backends that support insight/observability queries.
pub trait InsightBackend: Backend {
    async fn get_workflow_tree(&self, correlation_id: Uuid) -> Result<WorkflowTree>;
    async fn get_insight_stats(&self) -> Result<InsightStats>;
}
```

### Engine API

```rust
impl<D, B> Engine<D, B>
where
    D: Send + Sync + 'static,
    B: Backend,
{
    pub fn new(deps: D, backend: B) -> Self;
    pub fn with_handler(self, handler: Handler<D>) -> Self;

    /// Backend name for diagnostics/logging.
    pub fn backend_name(&self) -> &'static str;

    /// Preserved API shape: lazy dispatch future with fluent modifiers.
    pub fn dispatch<E>(&self, event: E) -> ProcessFuture<B>;
    pub fn dispatch_with_id<E>(&self, event_id: Uuid, event: E) -> ProcessFuture<B>;
    pub fn dispatch_to_workflow<E>(&self, correlation_id: Uuid, event: E) -> ProcessFuture<B>;

    /// Convenience for worker-only processes (blocking).
    pub async fn serve(self, shutdown: CancellationToken) -> Result<()>;

    /// Back-compat path for Runtime::start semantics (non-blocking handle).
    pub async fn start(&self, config: BackendServeConfig) -> Result<Runtime>;
}

// Capability-based methods
impl<D, B: WorkflowStatusBackend> Engine<D, B> {
    pub async fn get_workflow_status(&self, id: Uuid) -> Result<WorkflowStatus>;

impl<D, B: DeadLetterQueueBackend> Engine<D, B> {
    pub async fn list_dlq(&self, filters: DlqFilters) -> Result<Vec<DeadLetter>>;
    pub async fn retry_dlq(&self, event_id: Uuid, handler_id: String) -> Result<()>;
}

impl<D, B: InsightBackend> Engine<D, B> {
    pub async fn get_workflow_tree(&self, id: Uuid) -> Result<WorkflowTree>;
    pub async fn get_insight_stats(&self) -> Result<InsightStats>;
}
```

### ProcessFuture API (Preserved)

```rust
pub struct ProcessFuture<B: Backend> {
    // Internal fields
}

impl<B: Backend> ProcessFuture<B> {
    /// Set custom event ID (for idempotency).
    pub fn with_id(mut self, event_id: Uuid) -> Self;

    /// Set workflow/correlation ID.
    pub fn with_workflow_id(mut self, correlation_id: Uuid) -> Self;

    /// Set parent event ID (for causation tracking).
    pub fn with_parent(mut self, parent_id: Uuid) -> Self;

    /// Wait for terminal event matching closure.
    ///
    /// Closure returns:
    /// - None = not a terminal event, keep waiting
    /// - Some(Ok(value)) = success, return value
    /// - Some(Err(error)) = failure, propagate error
    pub fn wait<T, F>(self, matcher: F) -> WaitFuture<T, F, B>
    where
        F: Fn(&dyn Any) -> Option<Result<T>> + Send + 'static,
        T: Send + 'static,
        B: WorkflowSubscriptionBackend;
}

impl<B: Backend> IntoFuture for ProcessFuture<B> {
    type Output = Result<ProcessHandle>;
    // .await dispatches event and returns handle
}

pub struct ProcessHandle {
    pub event_id: Uuid,
    pub correlation_id: Uuid,
}

pub struct WaitFuture<T, F, B> {
    // Internal fields
}

impl<T, F, B> WaitFuture<T, F, B>
where
    F: Fn(&dyn Any) -> Option<Result<T>> + Send + 'static,
    T: Send + 'static,
    B: WorkflowSubscriptionBackend,
{
    /// Set timeout for waiting.
    pub fn timeout(mut self, duration: Duration) -> Self;
}

impl<T, F, B> IntoFuture for WaitFuture<T, F, B> {
    type Output = Result<T>;
    // .await returns matched value
}
```

### Backward Compatibility Contract

```rust
// v0.11 keeps these source-level contracts:
// - Engine::dispatch(...) -> ProcessFuture<_> (lazy)
// - ProcessFuture::with_id(...)
// - ProcessFuture::with_workflow_id(...)
// - ProcessFuture::with_parent(...)
// - ProcessFuture::wait(matcher) -> WaitFuture<T, F, B>
// - WaitFuture::timeout(duration)
// - ProcessHandle { event_id, correlation_id }
```

## Usage Examples

### 1. Basic Usage

```rust
use causal::{Engine, on};
use causal_postgres::PostgresBackend;
use tokio_util::sync::CancellationToken;

#[derive(Clone)]
struct Deps {
    db: PgPool,
    mailer: MailerClient,
}

#[derive(Clone, Serialize, Deserialize)]
struct OrderPlaced {
    order_id: String,
    customer_email: String,
}

#[derive(Clone, Serialize, Deserialize)]
struct OrderConfirmationSent {
    order_id: String,
}

#[tokio::main]
async fn main() -> Result<()> {
    let pool = PgPool::connect(&env::var("DATABASE_URL")?).await?;
    let deps = Deps {
        db: pool.clone(),
        mailer: MailerClient::new(),
    };

    // Create engine with PostgreSQL backend
    let engine = Engine::new(deps, PostgresBackend::new(pool))
        .with_handler(on::<OrderPlaced>().then(|event, ctx| async move {
            // Send confirmation email
            ctx.deps().mailer.send_email(
                &event.customer_email,
                "Order Confirmation",
                &format!("Order {} confirmed", event.order_id),
            ).await?;

            Ok(OrderConfirmationSent {
                order_id: event.order_id,
            })
        }));

    // Setup graceful shutdown
    let shutdown = CancellationToken::new();
    let shutdown_clone = shutdown.clone();

    tokio::spawn(async move {
        tokio::signal::ctrl_c().await.unwrap();
        println!("Shutdown signal received");
        shutdown_clone.cancel();
    });

    // Start serving (blocks until CTRL-C)
    engine.serve(shutdown).await?;

    Ok(())
}
```

### 3. Separate Dispatch and Serve Processes

```rust
// Process A: API server (dispatch only)
#[tokio::main]
async fn main() -> Result<()> {
    let pool = PgPool::connect(&env::var("DATABASE_URL")?).await?;
    let engine = Engine::new(deps, PostgresBackend::new(pool));

    // Axum/HTTP server
    let app = Router::new()
        .route("/orders", post(create_order))
        .layer(Extension(engine));

    axum::Server::bind(&"0.0.0.0:3000".parse()?)
        .serve(app.into_make_service())
        .await?;

    Ok(())
}

async fn create_order(
    Extension(engine): Extension<Engine<Deps, PostgresBackend>>,
    Json(payload): Json<CreateOrderRequest>,
) -> Result<Json<CreateOrderResponse>, StatusCode> {
    // Dispatch event
    let handle = engine.dispatch(OrderPlaced {
        order_id: Uuid::new_v4().to_string(),
        customer_email: payload.email,
    }).await.map_err(|_| StatusCode::INTERNAL_SERVER_ERROR)?;

    Ok(Json(CreateOrderResponse { event_id: handle.event_id }))
}

// Process B: Worker process (serve only)
#[tokio::main]
async fn main() -> Result<()> {
    let pool = PgPool::connect(&env::var("DATABASE_URL")?).await?;
    let engine = Engine::new(deps, PostgresBackend::new(pool))
        .with_handler(on::<OrderPlaced>().then(|event, ctx| async move {
            // Process order
            Ok(OrderProcessed { order_id: event.order_id })
        }));

    let shutdown = CancellationToken::new();
    setup_signal_handler(shutdown.clone());

    // Only serve, no dispatch
    engine.serve(shutdown).await?;

    Ok(())
}
```

### 4. Capability-Based Features

```rust
// Function that requires workflow status support
async fn wait_for_completion<D, B>(
    engine: &Engine<D, B>,
    correlation_id: Uuid,
) -> Result<()>
where
    D: Send + Sync + 'static,
    B: WorkflowStatusBackend,  // Compile error if backend doesn't support it
{
    loop {
        let status = engine.get_workflow_status(correlation_id).await?;

        if status.is_settled {
            println!("Workflow complete!");
            break;
        }

        tokio::time::sleep(Duration::from_millis(100)).await;
    }

    Ok(())
}

// Function that requires subscription support
async fn stream_workflow_events<D, B>(
    engine: &Engine<D, B>,
    correlation_id: Uuid,
) -> Result<()>
where
    D: Send + Sync + 'static,
    B: WorkflowSubscriptionBackend,  // Compile-time guarantee
{
    let mut events = engine.subscribe_workflow(correlation_id).await?;

    while let Some(event) = events.next().await {
        println!("Event: {} - {}", event.event_type, event.event_id);
    }

    Ok(())
}

// Function that requires DLQ support
async fn retry_failed_handlers<D, B>(
    engine: &Engine<D, B>,
) -> Result<()>
where
    D: Send + Sync + 'static,
    B: DeadLetterQueueBackend,
{
    let failed = engine.list_dlq(DlqFilters::default()).await?;

    for entry in failed {
        println!("Retrying: {} - {}", entry.event_id, entry.handler_id);
        engine.retry_dlq(entry.event_id, entry.handler_id).await?;
    }

    Ok(())
}

// Usage
let engine = Engine::new(deps, PostgresBackend::new(pool));

// These compile because PostgresBackend implements all capability traits
wait_for_completion(&engine, correlation_id).await?;
stream_workflow_events(&engine, correlation_id).await?;
retry_failed_handlers(&engine).await?;
```

### 5. Redis Backend (In-Memory Queue)

```rust
use causal::Engine;
use causal_redis::{RedisBackend, RedisConfig};

let config = RedisConfig {
    url: "redis://localhost:6379".to_string(),
    stream_name: "causal:events".to_string(),
    consumer_group: "causal-workers".to_string(),
    max_stream_length: 100_000,
    poll_interval: Duration::from_millis(100),
};

let engine = Engine::new(deps, RedisBackend::with_config(config))
    .with_handler(on::<OrderPlaced>().then(|event, ctx| async move {
        ctx.deps().process_order(&event).await?;
        Ok(OrderProcessed { order_id: event.order_id })
    }));

let shutdown = CancellationToken::new();
setup_signal_handler(shutdown.clone());

// Uses Redis Streams for queueing
engine.serve(shutdown).await?;
```

**Benefits:**
- ⚡ Fast in-memory queue
- 📊 Built-in monitoring via Redis CLI
- 🔄 Consumer groups for load balancing
- 💾 Persistence optional (AOF/RDB)

**Tradeoffs:**
- ❌ No workflow status queries (use PostgreSQL if needed)
- ❌ No DLQ (implement separately)
- ⚠️ Memory limits (not for huge event volumes)

### 8. Testing Patterns

```rust
use causal::Backend;
use tokio_util::sync::CancellationToken;

// In-memory backend for tests
struct InMemoryBackend {
    events: Arc<Mutex<Vec<DispatchedEvent>>>,
}

#[async_trait]
impl Backend for InMemoryBackend {
    async fn publish(&self, event: DispatchedEvent) -> Result<()> {
        self.events.lock().await.push(event.clone());
        Ok(())
    }

    async fn serve<D>(
        &self,
        executor: Arc<JobExecutor<D>>,
        _config: BackendServeConfig,
        shutdown: CancellationToken,
    ) -> Result<()> {
        // Process events immediately (no workers)
        while !shutdown.is_cancelled() {
            let event = {
                let mut events = self.events.lock().await;
                events.pop()
            };

            if let Some(event) = event {
                executor.execute(event).await?;
            } else {
                tokio::time::sleep(Duration::from_millis(10)).await;
            }
        }
        Ok(())
    }
}
```

## Breaking Changes

### Clean Break (No Deprecation Bridge)

v0.11.0 introduces breaking changes:

**Removed:**
- `Runtime::start(&engine, config)` → Use `engine.start(config)`
- `RuntimeConfig` → Use `BackendServeConfig`
- `Store` trait (public API) → Internal to backends
- `QueueBackend` trait → Internal to backends
- `Engine::builder()` → Use chaining `.with_handler()`
- `engine.queue_backend_name()` → Use `engine.backend_name()`

**Preserved:**
- `Engine::dispatch()` → Still returns `ProcessFuture`
- `ProcessFuture::with_id()` → Still works
- `ProcessFuture::with_workflow_id()` → Still works
- `ProcessFuture::with_parent()` → Still works
- `ProcessFuture::wait(matcher)` → Still works with full signature
- `WaitFuture::timeout()` → Still works
- `ProcessHandle { event_id, correlation_id }` → Still works
- All handler builders → Unchanged
- `Context` API → Unchanged

**Migration:**
```rust
// v0.10.x
let engine = Engine::new(deps, store);
let runtime = Runtime::start(&engine, RuntimeConfig {
    event_workers: 2,
    handler_workers: 4,
    ..Default::default()
}).await?;

// v0.11.0
let engine = Engine::new(deps, PostgresBackend::new(pool));
let runtime = engine.start(BackendServeConfig {
    event_workers: 2,
    handler_workers: 4,
    ..Default::default()
}).await?;
```

## Benefits

1. **Simpler API**: `engine.serve(shutdown)` vs `Runtime::start(&engine, config)`
2. **Better lifecycle**: Explicit `CancellationToken` for graceful shutdown
3. **Type safety**: Capability traits prevent runtime "not supported" errors
4. **Flexibility**: Backend abstraction enables Redis, NATS, Kafka, SQS
5. **Testability**: In-memory backends for fast unit tests
6. **Cleaner surface area**: Removes transitional shims and legacy API overlap

## Timeline

- **v0.11.0** (current): New backend API lands as a clean break
- **v0.11.x** (current cycle): Migration docs/examples and alternative backends
- **v0.12.0** (future): Stabilization and incremental improvements

## Questions for Review

1. Should `serve()` consume `self` or take `&self`?
   - **Decision**: Both patterns via separate APIs:
     - `Engine::serve(self, shutdown)` for simple worker binaries
     - `Engine::start(&self, config)` for non-blocking runtime semantics

2. Should we support `serve()` without `CancellationToken`?
   - **Decision**: Always require it for explicit lifecycle

3. Should capability traits be separate crates?
   - **Decision**: Keep in core, backends opt-in via implementation

4. Should we ship a temporary compatibility shim crate?
   - **Decision**: No. Keep the core API clean and provide a direct migration guide.
